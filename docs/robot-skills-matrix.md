# 机器人与 Skills 编排矩阵

## 定位

机器人是业务对象，Skills 是后台可复用能力。前台不直接暴露 Skills 给普通员工；员工只看到报销、合同协作、日程、项目等业务入口。业务入口提交到后端后先形成业务事件，再由事件规则触发对应机器人和 Skills。老板和总助可在机器人编排视图查看每个场景由哪些 Skills 组成、写回哪个系统、受哪些权限限制。

## 一期原则

- 不做聊天玩具，也不做“运行机器人”按钮；机器人由业务动作自动触发。
- 每个机器人至少包含：输入、处理 Skill、输出、写回对象、权限边界。
- Skills 可以复用，例如“会议纪要提取”既服务日程会议，也服务日报周报。
- 机器人名称中可以使用“助理/教练”，真实员工账号不使用“助理”作为身份。
- 合同、财务、员工画像等敏感数据按角色、组织、项目授权控制可见范围。

## 自动触发规则

| 业务动作 | 自动触发机器人 | 自动结果 |
|---|---|---|
| 上传发票图片 | 报销助理 | 识别发票字段，回填本地报销表格，生成待确认事项 |
| 上传合同/邮件材料 | 合同审批助理 | 创建合同项目组，生成风险备注，分派部门反馈 |
| 创建外勤/出差 | 差旅外勤助理 | 整理行程、外勤记录和后续报销提示 |
| 创建会议/预订会议室 | 日程和会议助理 | 协调会议室、日程、纪要和会后事项 |
| 提交日报 | 日报周报助理 | 自动汇总日报、量化工作量，低于阈值提醒主管 |
| 项目事项完成/延期 | 日报周报助理 | 自动汇总周报和月度量化提醒 |
| 报价草稿生成 | 报价助理 | 校验利润、有效期、赔付边界和审批要求 |

## 后端事件映射

| 后端事件 | 业务接口 | 机器人 | Skills |
|---|---|---|---|
| `invoice.uploaded` | `POST /api/expenses/invoices/autofill` | 报销助理 | 发票识别与命名 -> 报销规则校验 -> 项目费用归类 |
| `contract.uploaded` | `POST /api/contracts/projects` | 合同审批助理 | 合同材料读取 -> 合同风险分析 -> 合同项目组创建 -> 审批链路路由 -> 履约风险监控 |
| `meeting.created` | `POST /api/meetings` | 日程和会议助理 | 日程会议协调 -> 会议纪要提取 -> 访客接待引导 |
| `daily_report.submitted` | `POST /api/reports/daily` | 日报周报助理 | 日报周报汇总 -> 工作量化评分 -> 会议纪要提取 |

## 机器人清单

| 机器人 | 主要入口 | 核心 Skills | 输出/写回 | 一期边界 |
|---|---|---|---|---|
| 报销助理 | 报销、发票、报销审核 | 发票识别与命名、报销规则校验、项目费用归类 | 报销单、附件命名、费用归类 | 不自动付款，只给审批建议 |
| 差旅外勤助理 | 出差、外勤、日程 | 差旅行程整理、外勤签到校验、报销规则校验 | 差旅申请、外勤记录、异常提示 | 不代替主管审批 |
| 日报周报助理 | 我的待办、项目进度、工作偏离 | 日报周报汇总、工作量化评分、会议纪要提取 | 日报/周报/月度量化草稿 | 主管确认是否偏离计划 |
| 生意分析助理 | 关键项目、绩效结果、风险事项 | 生意指标查询、项目费用归类、报价风险校验 | 经营摘要、异常指标、风险事项 | 只展示授权汇总，不暴露明细 |
| 日程和会议助理 | 日程、会议室 | 日程会议协调、会议纪要提取、访客接待引导 | 会议日程、会议室预订、会后待办 | 不越权查看私密日程 |
| 访客教练 | 会议室、行政申请 | 访客接待引导、日程会议协调、AI 使用指导 | 访客记录、接待流程、行政协同 | 不开放内部敏感资料 |
| 产品培训教练 | 我的项目、项目进度 | 培训路径生成、测验与反馈、日报周报汇总 | 产品学习路径、测验结果 | 不替代主管评价 |
| 系统培训教练 | 权限申请、员工档案 | 培训路径生成、测验与反馈、AI 使用指导 | 系统学习路径、权限注意事项 | 按岗位权限展示 |
| AI教练 | AI 工作台、审批中心、事项中心 | AI 使用指导、日报周报汇总、生意指标查询 | 提示词、入口建议、注意事项 | 不直接绕过业务权限 |
| 报价助理 | 我的项目、关键项目 | 报价上下文收集、报价风险校验、审批链路路由 | 报价草稿、风险提示、审批建议 | 不直接对外发送报价 |
| 合同审批助理 | 合同协作项目、风险意见、合同归档 | 合同材料读取、合同风险分析、合同项目组创建、审批链路路由、履约风险监控 | 合同项目组、风险备注、正式审批、合同档案/履约监控 | 老板不上传合同，只处理终审/风险汇总 |

## 合同审批助理链路

```text
员工/主管提交合同或邮件材料
-> 合同材料读取
-> 合同风险分析
-> 合同项目组创建
-> 产品/财务/法务反馈沉淀
-> 审批链路路由
-> 正式审批
-> 合同档案与履约风险监控
```

## 开发落地建议

1. 后端维护 `skills` 表和 `robots` 表，机器人只引用 Skill ID。
2. 业务入口只提交业务动作，不直接调用机器人或单个 Skill。
3. 后端事件规则决定触发哪个机器人和哪些 Skills。
4. 每次事件执行写入运行日志：输入摘要、输出摘要、写回对象、操作者、模型版本或规则版本。
5. 高敏 Skill 增加权限校验和脱敏策略，例如合同风险、发票识别、人员画像。
6. Demo 阶段先用配置驱动，后续再接真实企微、财务、人事、项目、合同系统。
